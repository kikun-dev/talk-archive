# トークJSON抽出

チャットアプリのスクリーンショット（または画面録画）を読み取り、`/conversations/[id]/import` から取り込める
トークインポート JSON（`{ version, defaultYear, records[] }`）に変換する。

## 手順

### 1. 入力の準備

- 入力はチャットアプリのスクリーンショット画像（複数可、ユーザーが添付するか、パスで指定される）
- 画面録画（`.mp4` 等）が渡された場合は、先に ffmpeg で 1 秒間隔のフレームに分解する:
  ```
  ffmpeg -i <入力> -vf "fps=1" /tmp/frames/frame_%04d.png
  ```
  - 連続する類似フレーム（スクロール中の同一画面など）は読み飛ばしてよい。
    間引きすぎて会話の一部を取りこぼさないよう、内容が変化しているフレームは残す

### 2. メッセージの抽出ルール

各メッセージから以下を読み取る:

- `speaker`: 吹き出しのヘッダに表示されている名前
- `postedAt`: 表示されている日時（年の補完は次項参照）
- `type`: 種別判定
  - テキスト吹き出し → `text`
  - 写真のサムネイル → `image`
  - ▶ 再生ボタン付きサムネイル → `video`
  - シークバー + 再生時間の表示 → `audio`
- `content`:
  - `text` は本文をそのまま
  - `image` は添えられたキャプション/本文テキストがあれば入れる。なければ `null`
  - `video` / `audio` は常に `null`
- `title`: 常に `null`（本文と別にタイトルが読み取れる UI ではないため）
- `hasAudio`: `video` の場合のみ意味を持つ。音声ありと判別できれば `true`。
  ミュートアイコンなどで判別不能な場合は `true` を既定値とする（`text` / `image` / `audio` では `false` でよい）

### 3. 年の補完ルール（重要）

チャットアプリは以下の 2 通りの日時表記を使い分けている:

- **今年のトーク**: `M/D HH:MM`（年なし）
  - `--year` 引数（省略時はスクリーンショット撮影年 = 実行時点の年）で補完する
- **昨年以前のトーク**: `YYYY/MM/DD HH:MM`（年あり）
  - 表記の年をそのまま使う（補完不要）

出力 JSON の `postedAt` は ISO 8601 + `+09:00`（JST）形式で書く（例: `2026-07-10T21:15:00+09:00`）。
`defaultYear` には年なし表記の補完に使った年（`--year` の値、または既定の撮影年）を設定する。

### 4. 出力形式

本リポジトリのインポートスキーマ（`src/usecases/importUseCases.ts` の `TalkImportRecord` 参照）に従い、
JSON ファイルとして書き出す:

```json
{
  "version": 1,
  "defaultYear": 2026,
  "records": [
    {
      "speaker": "発言者名",
      "postedAt": "2026-07-10T21:15:00+09:00",
      "type": "text",
      "title": null,
      "content": "本文",
      "hasAudio": false
    }
  ]
}
```

- 出力先: `/tmp/talk-import-<日付>.json`（例: `/tmp/talk-import-20260711.json`）
- 書き出したら、ユーザーにファイルパスを提示する（インポート画面のファイル選択で使う）
- 実例は `docs/examples/talk-import-sample.json` を参照

### 5. 重複について

- スクロールスクショの重なりにより同一メッセージが複数枚に写り込むことがあるが、そのまま出力してよい。
  アプリ側（`buildRecordDedupKey`）が speaker + postedAt + 種別 + 本文先頭20文字で重複排除するため、
  抽出側で無理に重複を取り除く必要はない
- ただし、同一スクリーンショットを誤って二重に読み込んだ場合など、明らかな読み取りミスによる重複は省く

### 6. 品質チェック（出力前に必ず行う）

出力する前に、以下をユーザーに提示して確認を取る:

- 抽出件数（種別ごとの内訳もあると親切）
- 期間（`postedAt` の最小 〜 最大）
- 年の補完が意図どおりか（`--year` を使った場合は特に、今年のトークとして扱ってよいか）

明らかにおかしい件数・期間（例: 抽出漏れで数件しかない、日時が未来すぎる等）がある場合は、
書き出す前にユーザーに再確認する。

## 完了条件

- `/tmp/talk-import-<日付>.json` がスキーマ準拠で書き出されている
- 件数・期間・年補完の要約をユーザーに提示し、確認を取っている
- ファイルパスをユーザーに提示している（インポート画面 `/conversations/[id]/import` で選択する旨も案内する）
