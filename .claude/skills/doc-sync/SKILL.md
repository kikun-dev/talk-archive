# ドキュメント同期

コードの実態とドキュメントのズレを検出し、修正する。Claude / Codex 共用スキル。

## 対象範囲の特定

- PR 番号指定: `gh pr view <N>` + `gh pr diff <N>` でその PR の変更を対象にする
- コミット範囲指定: `git diff <範囲> --stat` で対象を把握する
- 省略時: カレントブランチと `main` の差分に加えて、直近のマージ済み PR 5件
  （`gh pr list --state merged --limit 5`）のタイトル・差分概要を対象にする

## 同期チェックマップ（この表の左が変わったら右を確認する）

| コード側の変更 | 確認・更新するドキュメント |
|---|---|
| 機能の追加・変更・削除 | `docs/requirements.md` の該当セクション、`docs/roadmap.md` の該当フェーズ |
| テーブル・migration・Storage の追加/変更 | `docs/database.md`（テーブル・record 属性・ストレージ設計） |
| 層構成・依存関係の変更 | `docs/architecture.md`、`rules/architecture.md` |
| 開発フロー・CI・検証手順の変更 | `docs/development.md`、`README.md` |
| ADR 追加 | 発端 Issue の Decision 欄との相互参照 |
| フェーズ/計画の進捗（Issue クローズ・PR マージ） | `docs/roadmap.md` の該当フェーズ表（`✅ PR #N` 形式） |
| 監査指摘への対応 | `docs/audits/` の該当ドキュメントの Issue 対応表 |
| ルール・運用の変更 | `rules/` ↔ `CLAUDE.md` / `AGENTS.md` / `docs/ai/ai-collaboration.md` の整合（**マスタは rules/ と docs/ai/**。役割別ファイルには複製せず参照させる） |
| 定型手順の変更 | `.claude/skills/` / `.codex/skills/` の該当スキルの手順（共用スキルは `.codex/skills/` からのシンボリックリンクで共有されているため、実体は `.claude/skills/` 側を編集する） |

## 追加チェック（差分に関わらず毎回）

- Issue に「Decision を残す」とされた変更（新規ライブラリ・層構造・データモデル・重要 UX）が
  ADR 昇格（`/adr` の昇格基準）に該当しないか
- ドキュメント内の相互参照（Issue 番号・ファイルパス・ADR 番号）が実在するか

## 修正の実施

- ズレを一覧化してから修正する（何を根拠にズレと判断したかを添える）
- 事実の更新（済/未済、番号、一覧への追加）はそのまま修正する
- **意味・方針を変える修正**（運用メモの書き換え、ルールの変更など）は、修正案を提示して
  ユーザーの確認を取ってから行う
- 修正はドキュメントのみ。コード側を「ドキュメントに合わせて」変更しない
  （コードが正でドキュメントが古い、が原則。逆と思われる場合は報告して確認する）

## 完了条件

- 検出したズレがすべて「修正済み」か「要確認として報告済み」になっている
- 修正内容のサマリー（ファイルごとの変更理由）を提示している
